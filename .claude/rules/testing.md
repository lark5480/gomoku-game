# 测试规则

- **测试文件位置**: 所有测试文件放在 `tests/` 目录中
- **测试命名**: 测试文件使用 `.test.js` 或 `.test.mjs` 后缀
- **测试结构**: 每个测试文件应包含清晰的测试用例分组
- **测试覆盖**:
  - 核心游戏逻辑（棋盘状态、移动验证、胜利检测）
  - 边界情况（棋盘边缘、无效输入）
  - 用户交互（点击处理、游戏流程）
- **测试执行**: 使用 `node test.mjs` 运行游戏逻辑测试
- **手动测试**: 在浏览器中测试 UI 响应性和游戏流程
- **测试数据**: 使用有意义的测试数据，覆盖典型场景和边缘情况
- **断言**: 测试应有清晰的通过/失败输出，方便调试
