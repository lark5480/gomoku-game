# 测试规则

- **测试文件位置**: 所有测试文件放在 `tests/` 目录中
- **测试命名**: 测试文件使用 `.test.js` 或 `.test.mjs` 后缀
- **测试结构**: 每个测试文件应包含清晰的测试用例分组
- **测试覆盖**:
  - 棋盘逻辑（状态、移动验证、胜利检测、撤销）
  - AI 棋型评估（活三/冲四/不连续/双面/杀棋检测）
  - 边界情况（棋盘边缘、无效输入）
- **测试执行**: `npm test` 运行全部测试，`node tests/test.mjs` 仅 Board 测试，`node tests/ai.test.mjs` 仅 AI 测试
- **手动测试**: 浏览器中验证 UI 响应性、AI 对战表现
- **测试数据**: 使用有意义的测试数据，覆盖典型场景和边缘情况
- **断言**: 测试应有清晰的通过/失败输出，方便调试
