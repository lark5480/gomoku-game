# API 约定（未来扩展参考）

**注意**: 当前项目为纯前端游戏，无后端API。以下规则供未来添加API功能时参考。

- **RESTful 设计**: 使用 RESTful 风格，资源名使用复数形式
- **响应格式**: 统一响应格式：`{ data, error, meta }`
  - `data`: 主要响应数据
  - `error`: 错误信息（成功时为 null）
  - `meta`: 元数据（分页信息、时间戳等）
- **HTTP 状态码**: 错误码遵循 HTTP 标准语义
  - 200: 成功
  - 400: 客户端错误（无效请求）
  - 401: 未认证
  - 403: 无权限
  - 404: 资源不存在
  - 500: 服务器内部错误
- **API 文档**: 所有接口需要在 OpenAPI/Swagger 文档中声明
- **分页**: 分页参数统一使用 `page` / `page_size`
- **版本控制**: API 版本通过 URL 前缀或请求头控制（如 `/api/v1/`）
- **认证**: 使用 JWT token 进行用户认证
